# WhiskerShelf Idea Spark Brief — Schema Reference

`brief.md` is generated by WhiskerShelf's Idea Spark. The LLM is asked to produce a specific structure; this document is the contract you can rely on when parsing the brief.

## Top-level structure

```markdown
# 灵感碰撞：<paper 1> × <paper 2> [× <paper 3>] [× <paper 4>]

> 生成时间: <YYYY-MM-DD HH:MM>
> 选中论文: <N> 篇
> 用户补充上下文: <text or "无">

## 1. 共同主题与方法张力
...

## 2. 研究方向

### 方向 1: <title>
**核心 Idea**: <one-sentence pitch>
**方法迁移路径**: ...
**预期难点**: ...
**验证方案**: ...
**可能的跨域跳跃**: ...

### 方向 2: <title>
...

### 方向 3: <title>
...
(3–5 directions total)

## 3. 跨域跳跃建议
...

## 4. 风险与盲点
...
```

## Field-by-field guarantee

For each direction block, the LLM is prompted to produce all five fields. In practice:

| Field | Always present? | What to expect |
|---|---|---|
| 核心 Idea | yes | 1–2 sentences; the elevator pitch |
| 方法迁移路径 | yes | Names a source paper/method and an adaptation step |
| 预期难点 | yes | Lists 1–4 failure modes or open questions |
| 验证方案 | yes | Concrete experiment: dataset + metric + baseline |
| 可能的跨域跳跃 | often | Suggests 1–2 adjacent application areas |

If any of the first four is missing, the brief is **incomplete** — surface this to the user before proceeding.

## Why headings matter (for `parse_brief.py`)

`scripts/parse_brief.py` parses the brief by regex-matching the canonical Chinese headings. If the brief deviates (e.g., the LLM uses "Idea" instead of "核心 Idea"), the parser will leave the field as `null` and print a warning. Do not silently trust the parse — re-read the original if a field is missing.

## Reasoning / chain-of-thought

When WhiskerShelf's "thinking mode" is enabled, the model's chain-of-thought is also written into `brief.md` as a fenced ` ```text ` block under a `## 思考过程` heading. This is **metadata for you**, not for the user — quote 1–2 lines from it if the user's question is "what was the model thinking?", but don't paste it wholesale.

## Common variations to be robust to

- Direction count: 2–5 (brief always has 3–5; tolerate 2 in edge cases).
- Paper count in title: 2–4.
- Bold/italic markdown may be missing on some fields.
- The 跨域跳跃 field may be merged into the direction block or appear at the end of the document under section 3.
- The risk section may be labeled 风险 / 风险与盲点 / Risks — accept any of them.
